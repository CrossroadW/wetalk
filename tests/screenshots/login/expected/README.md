# 登录界面截图测试说明

本目录包含登录界面的预期效果图，用于自动化截图对比测试。

## 测试图片说明

### 01_qr_code_initial.png
**状态**: 二维码登录初始状态
**描述**:
- 显示微信登录标题
- 显示二维码（等待扫描）
- 显示提示文字"请使用微信扫描二维码登录"
- 二维码未过期，未被扫描

**如何进入此状态**:
1. 启动应用
2. 本地没有保存的 token（首次登录）
3. 自动显示二维码登录界面

---

### 02_qr_code_loading.png
**状态**: 二维码加载中状态
**描述**:
- 显示加载动画或提示
- 二维码正在生成或网络请求中
- 可能显示"正在加载..."文字

**如何进入此状态**:
1. 启动应用
2. WebSocket 连接建立中
3. 发送 `qr_login_init` 请求
4. 等待服务器响应期间

---

### 03_qr_code_scanned.png
**状态**: 二维码已扫描，等待确认
**描述**:
- 二维码变为已扫描状态（可能显示头像或勾选标记）
- 显示"扫描成功，请在手机上确认登录"
- 等待用户在手机端输入用户名并确认

**如何进入此状态**:
1. 用户使用手机扫描二维码
2. 手机端打开登录页面
3. 尚未输入用户名或点击确认
4. 桌面端收到扫描通知（如果实现了中间状态推送）

---

### 04_direct_login.png
**状态**: 直接登录状态（已有 token）
**描述**:
- 显示"进入微信"按钮
- 显示用户头像或用户名
- 不显示二维码
- 用户可以直接点击按钮进入主界面

**如何进入此状态**:
1. 启动应用
2. 本地已保存有效的 token
3. 发送 `verify_token` 请求验证成功
4. 显示直接登录按钮

---

## 测试流程

### 自动化测试
运行 `LoginScreenTest` 会自动生成以上 4 种状态的截图，保存到 `output/` 目录。

### 对比方式
- 使用像素对比或感知哈希算法
- 允许小幅度差异（如时间戳、动画帧）
- 主要验证布局、颜色、控件位置

### 更新预期图片
如果界面设计有意更改：
1. 运行测试生成新截图
2. 人工确认新截图符合设计
3. 将 `output/` 中的图片复制到 `expected/`
4. 提交到 git

## 注意事项

- 预期图片（expected/）提交到 git
- 生成图片（output/）被 git 忽略
- 截图分辨率可以适当降低以节省存储
- 建议使用 800x600 或更小的窗口尺寸
