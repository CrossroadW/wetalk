# 错误处理

## 通用错误响应

所有失败的响应都遵循以下格式：

```json
{
  "type": "原始请求类型",
  "success": false,
  "data": {
    "message": "错误描述"
  }
}
```

## 常见错误信息

### 认证相关

- `"未登录"` - 需要先验证 token
- `"缺少 token"` - verify_token 请求缺少 token 字段
- `"token 无效"` - token 已过期或不存在

### 二维码登录相关

- `"缺少参数"` - 请求缺少必填字段
- `"二维码已失效"` - 二维码会话不存在
- `"二维码已过期"` - 二维码会话已超时（超过 5 分钟）
- `"用户不存在"` - 指定的用户名不存在

### 通用错误

- `"未知消息类型: xxx"` - 服务器不支持该消息类型

## 未知消息类型

如果客户端发送了服务器不支持的消息类型：

```json
{
  "type": "error",
  "success": false,
  "data": {
    "message": "未知消息类型: xxx"
  }
}
```

## 错误处理最佳实践

### 后端

1. **验证必填字段**: 检查所有必填字段是否存在
2. **返回明确错误**: 错误信息应该清晰描述问题
3. **统一格式**: 所有错误都使用 `success: false` 格式
4. **记录日志**: 记录错误信息便于调试

### 前端

1. **检查 success 字段**: 始终检查响应的 `success` 字段
2. **显示错误信息**: 将 `data.message` 显示给用户
3. **处理特殊错误**:
   - `"未登录"` 或 `"token 无效"` → 跳转到登录页面
   - `"二维码已过期"` → 重新生成二维码
4. **重试机制**: 对于网络错误，可以实现自动重试
