# 认证流程与实现要求

## 认证流程

### 首次登录（二维码）

1. 客户端连接 WebSocket (`ws://localhost:8000/ws`)
2. 客户端发送 `qr_login_init` 请求
3. 服务器返回 `session_id` 和 `qr_url`
4. 客户端使用 zxing 生成二维码显示 `qr_url`
5. 用户扫码，浏览器打开网页，输入用户名
6. 服务器推送 `qr_confirmed` 给客户端
7. 客户端保存 `token` 到本地文件（使用 AppPaths 获取数据目录）
8. 客户端进入主界面

### 已登录（Token 验证）

1. 客户端连接 WebSocket
2. 客户端从本地文件读取 `token`
3. 客户端发送 `verify_token` 请求
4. 服务器验证 token 并返回用户信息
5. 客户端直接进入主界面

### 登出

1. 客户端发送 `logout` 请求
2. 服务器清除 token
3. 客户端删除本地 token 文件
4. 客户端返回登录页面

## 实现要求

### 后端要求

1. **严格遵守消息格式**: 所有响应必须包含 `type`, `success`, `data` 字段
2. **认证检查**: 除了 `qr_login_init` 和 `verify_token`，其他请求都需要验证 token
3. **错误处理**: 所有错误都应返回 `success: false` 和错误信息
4. **数据验证**: 验证所有必填字段是否存在
5. **推送通知**: 实现 `qr_confirmed` 推送
6. **连接管理**: 维护 WebSocket 连接状态，处理断线重连

### 前端要求

1. **严格遵守消息格式**: 所有请求必须包含 `type` 和 `data` 字段
2. **异步处理**: 使用回调或 Promise 处理响应
3. **推送处理**: 注册推送消息处理器（如 `qr_confirmed`）
4. **错误处理**: 检查 `success` 字段，处理错误情况
5. **Token 管理**:
   - 保存 token 到本地文件
   - 启动时读取 token
   - 登出时删除 token
6. **二维码生成**: 使用 zxing-cpp 生成二维码图片
7. **自动重连**: WebSocket 断线后自动重连

## Token 持久化

### 保存位置

使用 `AppPaths::instance().dataDir()` 获取数据目录，保存为 `token.txt`。

### 保存格式

纯文本文件，只包含 token 字符串，无其他内容。

### 安全性

当前实现不加密 token（适合演示），生产环境应考虑加密存储。

## WebSocket 连接管理

### 连接状态

- **未连接**: 初始状态
- **连接中**: 正在建立连接
- **已连接**: 连接成功，可以发送消息
- **断开**: 连接断开，需要重连

### 重连策略

- 断线后等待 5 秒自动重连
- 最多重试 3 次
- 重连成功后重新验证 token

### 心跳机制

可选：每 30 秒发送一次心跳消息，保持连接活跃。
